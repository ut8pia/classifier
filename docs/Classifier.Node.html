Class: Node

Classifier.Node(key, ninopt, noutopt, childrenopt)

Each Node is responsible for storing a component of the array and keeping track of the count of arrays that either conclude at the node or traverse through it. Its properties:

  • key
  • nin
  • nout
  • children

serve this purpose.

The children of the Node can take two forms: they can be organized as an ArrayQueue or as a SortedArray of Node objects. The Classifier utilizes a sorter to decide how the Node's children should be arranged. This arrangement can either be based on the order of insertion, which is achieved using the ArrayQueue, or it can be determined by a custom comparator when utilizing the SortedArray.

The Node class provides two essential methods:

  • n()
  • getChild(key)

respectively counting the overall number of paths sharing the node and addressing a child.

Methods

getChild(key) → {Array.<Number, Classifier.Node>}

This method retrieves the child node of this node based on a given key. The result returned is a pair [index, child], where index represents the position of the child among all the children of the current node.

The method of searching for the child depends on how the children are stored. If the children are stored in a SortedArray, a logarithmic search is employed. If the children are stored in a ArrayQueue, the check is performed using identity among keys. This entails iterating through all the children to find the matching key.

It's important to note that an index is always returned, even when the specified child is not found. In such cases, the index indicates the position where the child should be located.

(static) of(key, keyComparator) → {Classifier.Node}

All that is needed to instantiate a Node is a key and a comparator.

The comparator is expected to be a 'key' comparator. The responsibility of converting this 'key' comparator into the suitable Node comparator lies with the Classifier. The resulting Node comparator is passed to the constructor of a SortedArray which is intended for storing the children of the returned Node.

If a comparator is not provided, the children of the returned Node are sorted by order of insertion within an ArrayQueue.
