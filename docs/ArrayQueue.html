Class: ArrayQueue

ArrayQueue(fifoopt, itemsopt)

The most straightforward way to organize items in a queue is by placing them in the order they were added.

In the context of the ArrayQueue, newly added items are always placed at the end of the internal array:

  • items

However, there's a Boolean attribute:

  • fifo

that determines whether the order of extraction from the queue either follows the order of insertion or reverses it.

Remarkably, each item within the ArrayQueue is unique and cannot be considered equivalent to any other item, even to itself. The same identical item can be added multiple times to the same ArrayQueue.

Consistently, the methods:

  • has()
  • remove()

from the Collection interface always return false.

For optimal utilization, the ArrayQueue is most suitable for tasks primarily involving the

  • add()
  • peek()
  • poll()

methods. The execution of the "search and selection pattern" is among them. Therefore, the ArrayQueue is a suitable choice for such a purpose.

Constructor

new ArrayQueue(fifoopt, itemsopt)

The ArrayQueue is a straightforward Queue that employs a couple of properties:

  • fifo
  • items

Optionally, these properties can be passed to the ArrayQueue constructor.

search(space, max) → {Each}

Explores the given space by using this Queue as a stack.

In case fifo === false the order of the contiguous items returned by the space is inverted, to emulate a depth-first order exploration.
