Class: Classifier

Classifier(sorteropt, rootopt)

The Classifier is a Collection of arrays. Each arrays is stored within a tree structure in the form of a path of Nodes.

Unlike traditional collections, however, the Classifier has the unique capability to store multiple instances of the same array while keeping track of the overall count of occurrences. This distinct functionality has implications for certain methods in the Collection interface, such as:

  • n()
  • add()

as they can now accept optional arguments to manage the multiple occurrences of arrays efficiently.

Furthermore, the Classifier's tree-like structure allows it to provide hierarchical methods:

  • class()
  • space()
  • view()
  • views()

Out of these methods, the most significant ones are the class() and view() methods.

A helpful way to conceptualize the Classifier is to think of it as a Map. Unlike a traditional Map where entries are limited to couples [key, value] and only one entry can be associated to each key, here:

  • The entries can consist of arrays of varying lengths
  • Multiple entries can share same key
  • Each component of the array can serve as key.

However, the ultimate goal for the Classifier is to facilitate the implementation of representation-based equivalences among objects. You can delve into these concepts in-depth, complete with example code, by referring to the booklet:

Venditti, R. (2023). Classifier.js: Idea, Design, and Use Cases. Amazon KDP

Constructor

new Classifier(sorteropt, rootopt)

The Classifier is a Queue of arrays that employs a pair of properties:

  • sorter
  • root

to collect arrays in an internal tree structure.

Optionally, these properties can be passed to the Classifier constructor.

A further properties:

  • defaultView

is implicitly created by the constructor.

sorter

A function(path, key) returning a comparator.

This function plays a crucial role in the context of the Classifier. Its primary purpose is to determine the comparator used when creating a new Node. This comparator is intended to sort the children of the newly created node in a SortedArray.

When the function is called, it takes two arguments:

  • the path argument, which is a path of entries [index, child] addressing the Node to be created
  • the key argument, which is the key to be stored in the new Node.

In cases where the sorter function doesn't provide a comparator, the Classifier defaults to sorting the children of the new node based on the order of their insertion, using an ArrayQueue.

It's worth noting that the expected comparator is a key comparator, not a Node comparator. The Classifier is tasked with converting this key comparator into the appropriate Node comparator, which is subsequently passed to the SortedArray constructor.

class(…keys) → {Each.<Array.<*>>}

Retrieves all arrays containing the specified keys by calling the corresponding class(...keys) methods of its defaultView. Consequently, are returned absolute arrays.

It's crucial to highlight that the provided keys are not required to be heading keys. In essence, some of the provided keys may be undefined.

Example
const 
    classifier = new Classifier()
        .let('a', 0)
            .let('b', 1),
                .let('c', 1),
    cls = classifier.class(undefined, 1);

assert.deepEqual(cls.toArray(), [['b', 1], ['c', 1]])

n(keysopt)

Because the Classifier can store multiple occurrences of the same array, the methods n() from the Collection interface can accommodate an optional item argument. When you provide this argument, it limits the count to the number of occurrences of the specified item.

space(…keys) → {What}

The primary purpose of this method is to provide formal definition of the search space that encompasses all paths of Nodes containing the specified keys.

The method is designed to handle scattered keys as input. If you don't provide any keys, it will by default return the space of the entire Classifier, which can still be quite valuable. You have the flexibility to merge the returned space with other spaces or fine-tune it using custom predicates to suit your needs.

view(directopt, keys) → {Classifier.View|undefined}

An instance of the class View representing the subtree addressed by the given keys. The View provides all the functionalities of a Classifier.

Nonetheless, the View is merely an interface. Every modification of the View directly affects the main Classifier tree.
